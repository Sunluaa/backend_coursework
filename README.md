# Oprosnik

Oprosnik - веб-сервис для создания и прохождения опросов. Он задуман как рабочий инструмент для небольших команд, учебных проектов и внутренних исследований: быстро собрать форму, раздать ее нужной аудитории, а потом посмотреть ответы без ручной обработки таблиц.

Главная идея проекта - дать автору контроль над жизненным циклом опроса. Пока опрос в черновике, его можно спокойно настраивать: менять описание, добавлять вопросы, редактировать варианты ответа и переставлять вопросы местами. После публикации структура фиксируется, чтобы результаты не ломались из-за поздних правок. Когда сбор ответов завершен, опрос можно закрыть и при необходимости удалить.

## Что Умеет Сервис

### Опросы и доступ

Автор создает опрос, задает название и описание, а затем выбирает режим доступа:

- публичный доступ - опрос появляется в общем списке на сайте;
- доступ по ссылке - опрос не виден в публичном списке, но открывается по прямой ссылке;
- анонимные ответы - незарегистрированные пользователи могут голосовать;
- повторные ответы - авторизованный пользователь может пройти один и тот же опрос больше одного раза.

По умолчанию эти настройки выключены. Такой вариант безопаснее: новый опрос сначала закрыт от случайной аудитории, а автор сам решает, когда и кому его открыть.


## Роли

Гость может открыть главную страницу, смотреть публичные опросы и проходить те опросы, где разрешены анонимные ответы.

Зарегистрированный пользователь может создавать свои опросы, редактировать черновики, публиковать и закрывать опросы, смотреть результаты и удалять свои опросы.

Администратор управляет данными через стандартную Django admin.

### Работа автора

В личном разделе автор видит свои опросы, статусы, количество вопросов и количество прохождений. Для черновиков доступны редактирование и публикация, для опубликованных опросов - просмотр результатов, ссылка на прохождение и закрытие. Закрытые опросы можно удалить.

Вопросы в черновике добавляются в естественном порядке: первый созданный вопрос идет первым, следующий - после него. При необходимости порядок можно изменить перетаскиванием.

### Типы вопросов

В опросе поддерживаются четыре типа вопросов:

- один вариант ответа;
- несколько вариантов ответа;
- текстовый ответ;
- оценка по шкале 1-5 или 1-10.

Для вариантов ответа есть встроенный редактор: можно добавлять поля, оставлять пустые строки, удалять лишнее. Пустые варианты не сохраняются. Для вопросов с выбором нужен минимум один вариант на этапе редактирования и минимум два варианта для публикации готового опроса.

Оценка на странице прохождения показывается как ползунок. Это удобнее, чем ряд радиокнопок, особенно для шкалы на 10 пунктов.

### Прохождение опросов

Респондент открывает опубликованный опрос, указывает имя при необходимости и отвечает на вопросы. Сервис проверяет обязательные вопросы, не принимает варианты из чужих вопросов и не дает отправить несколько ответов на один и тот же вопрос.

Если анонимные ответы выключены, пройти опрос может только авторизованный пользователь. Если повторные ответы выключены, зарегистрированный пользователь не сможет отправить второй ответ на тот же опрос.

После отправки пользователь попадает на страницу благодарности. Результаты ему не показываются: они доступны только автору.

### Результаты и статистика

Страница результатов собирает ответы по каждому вопросу:

- для одиночного выбора показывается количество голосов и доля ответов;
- для множественного выбора показывается количество выборов и доля прохождений, в которых был выбран конкретный вариант;
- для текстовых вопросов выводится список текстовых ответов;
- для рейтинга показываются средняя оценка и распределение по шкале.

Пример для множественного выбора: если опрос прошли 3 раза, первый вариант выбрали все 3 раза, второй - 2 раза, а третий - 1 раз, статистика покажет 100%, 66.7% и 33.3%. Это отражает не долю среди всех галочек, а реальную частоту выбора варианта среди прохождений.

## Интерфейс

Проект использует обычные Django Templates и Bootstrap, но интерфейс доработан под задачу: навигация разделена на группы, карточки имеют единый размер, футер прижат к низу страницы, есть светлая и темная тема. Для фона используется декоративная геометрическая графика в зеленой палитре приложения.

Основные страницы:

- главная страница с последними публичными опросами;
- регистрация, вход и личный кабинет;
- панель опросов;
- список "Мои опросы";
- создание и редактирование опроса;
- создание и редактирование вопросов;
- публичный список опросов;
- прохождение опроса по UUID-ссылке;
- результаты опроса для автора.

## Техническая Часть

### Стек

- Python 3.13
- Django 5.2
- Django REST Framework
- PostgreSQL
- Django Templates
- Bootstrap 5
- стандартная Django Auth
- drf-spectacular и Swagger UI
- Docker Compose
- Gunicorn для контейнера
- Waitress для Windows-запуска без Docker
- Django TestCase

### Архитектура

Проект построен на Django MVT. HTML-интерфейс и REST API используют одну и ту же бизнес-логику, чтобы правила публикации, прохождения и расчета результатов не расходились между веб-страницами и API.

Ключевые части:

- `apps/accounts` - регистрация, вход, выход и личный кабинет;
- `apps/core` - главная страница;
- `apps/surveys/models.py` - модели опросов, вопросов, вариантов, прохождений и ответов;
- `apps/surveys/forms.py` - HTML-формы;
- `apps/surveys/views.py` - HTML-представления;
- `apps/surveys/services.py` - бизнес-правила и расчет статистики;
- `apps/surveys/selectors.py` - переиспользуемые выборки и агрегаты;
- `apps/surveys/api_views.py` - DRF endpoint'ы;
- `apps/surveys/serializers.py` - сериализация для API;
- `apps/surveys/permissions.py` - права доступа API;
- `templates` - общие шаблоны;
- `static` - стили, JavaScript и фоновые изображения.

### Модель данных

Основные сущности:

- `Survey` - опрос со статусом `draft`, `published` или `closed`;
- `Question` - вопрос внутри опроса;
- `Choice` - вариант ответа для вопросов с выбором;
- `SurveyResponse` - одно прохождение опроса;
- `Answer` - ответ на конкретный вопрос в рамках прохождения.

Статусы опроса важны для бизнес-логики. Черновик можно редактировать. Опубликованный опрос можно проходить и закрывать. Закрытый опрос больше не принимает ответы.

## Запуск

### Переменные окружения

Создайте файл `.env` в корне проекта:

```env
DJANGO_SECRET_KEY=change-me
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1,0.0.0.0

POSTGRES_DB=oprosnik_db
POSTGRES_USER=oprosnik_user
POSTGRES_PASSWORD=oprosnik_password
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
```

Для Docker нужно использовать хост базы `db`, потому что так называется сервис PostgreSQL в `docker-compose.yml`:

```env
POSTGRES_HOST=db
```

### Docker

Перед запуском убедитесь, что Docker Desktop запущен и работает Linux engine.

```bash
docker compose up --build
```

В другом терминале примените миграции и создайте администратора:

```bash
docker compose exec web python manage.py migrate
docker compose exec web python manage.py createsuperuser
```

После запуска:

- `http://127.0.0.1:8000/` - сайт;
- `http://127.0.0.1:8000/admin/` - Django admin;
- `http://127.0.0.1:8000/api/docs/` - Swagger UI.

## Тесты

```bash
python manage.py test
```

Во время тестов проект использует SQLite test database. Это ускоряет проверку бизнес-правил и API и не требует поднятого PostgreSQL.

Тестами покрыты ключевые сценарии: публикация опроса, запрет редактирования чужих данных, прохождение гостем, запрет анонимного прохождения, порядок вопросов, удаление закрытого опроса, API-отправка ответа и расчет статистики.

## REST API

API находится под префиксом `/api/`.

Публичные endpoint'ы:

- `GET /api/surveys/public/`
- `GET /api/surveys/public/<uuid>/`
- `POST /api/surveys/public/<uuid>/submit/`

Endpoint'ы автора:

- `GET /api/surveys/my/`
- `POST /api/surveys/my/`
- `GET /api/surveys/my/<id>/`
- `PATCH /api/surveys/my/<id>/`
- `DELETE /api/surveys/my/<id>/`
- `POST /api/surveys/my/<id>/publish/`
- `POST /api/surveys/my/<id>/close/`
- `GET /api/surveys/my/<id>/results/`
- `POST /api/questions/`
- `PATCH /api/questions/<id>/`
- `DELETE /api/questions/<id>/`
- `POST /api/choices/`
- `PATCH /api/choices/<id>/`
- `DELETE /api/choices/<id>/`

Документация API:

- `/api/schema/` - OpenAPI schema;
- `/api/docs/` - Swagger UI.
